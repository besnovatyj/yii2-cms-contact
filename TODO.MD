Итог рефакторинга

1. Переименование `Contact` → `Message`

- Сущность `Contact` → `Message` (таблица `contact_mails`), добавлены поля `subject`, `send_status`, `send_method`
- Новая сущность `Contact` — адресная книга (таблица `contacts`)
- Событие `ContactMessageSent` → `MessageSent` (старый класс оставлен как `@deprecated` алиас)

2. Каналы отправки (TODO #3, #5)

- `ChannelInterface` — `getType()`, `canHandle(sendMethod)`, `send(Message)`
- `EmailChannel` и `TelegramChannel` — реализации без хардкода, всё из конфига модуля
- Добавить новый канал: создать класс + строчка в `dependencies.php`

3. Очередь и статусы (TODO #4)

- `Message` сохраняется с `send_status = pending`
- `MessageSentListener` решает: либо кладёт `SendMessageJob` в очередь (если нужна внешняя отправка), либо сразу ставит `no_send`
- `SendMessageJobHandler` вызывает `MessageSender` и обновляет статус на `sent`/`failed`

4. Фабрика/оркестратор (TODO #5)

- `MessageSender` читает `send_method` из конфига и передаёт сообщение только тем каналам, у которых `canHandle()` = `true`
- DIC-конфиг в `dependencies.php` — один файл, без хардкода в Bootstrap

5. Адресная книга (TODO #1)

- CRUD через `backend/ContactController` + views (create, update, view, index)
- `AddressBookService::syncFromMessage()` автоматически сохраняет данные отправителя при каждом новом сообщении

6. Отправка из админки (TODO #2)

- Кнопка «Написать» на каждом контакте в адресной книге
- HTMX: форма открывается в `#send-message-panel` без перезагрузки страницы
- Партиал `_send-message-form.php` — используется и для HTMX, и для полной страницы
